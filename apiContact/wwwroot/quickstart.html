Three Steps to First Message

Demo credentials (alice / password123) are seeded automatically — no account setup needed

1

Login and receive your JWT access token

// POST /api/auth/login (demo credentials work out of the box) const res = await fetch("/api/auth/login", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ usernameOrEmail: "alice", password: "password123" }) }); const { data } = await res.json(); const token = data.accessToken; // keep this — used in all protected requests
2

Connect to the SignalR WebSocket hub

// npm install @microsoft/signalr import * as signalR from "@microsoft/signalr"; const conn = new signalR.HubConnectionBuilder() .withUrl(`/hubs/chat?access_token=${token}`) .withAutomaticReconnect() .build(); conn.on("ReceiveMessage", msg => console.log(`[${msg.senderName}] ${msg.content}`)); await conn.start(); await conn.invoke("JoinRoom", "<room-guid>");
3

Send a message — auto-broadcast to all room members

// POST /api/messages — triggers ReceiveMessage on all connected clients in the room await fetch("/api/messages", { method: "POST", headers: { "Content-Type": "application/json", "Authorization": `Bearer ${token}` }, body: JSON.stringify({ roomId: "<room-guid>", content: "Hello, world!" }) });

Token Management

Access tokens are short-lived — use the refresh flow to renew silently without prompting the user

→

Silently refresh an expiring access token

// POST /api/auth/refresh — rotates both tokens const r = await fetch("/api/auth/refresh", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ refreshToken: data.refreshToken }) }); const renewed = (await r.json()).data; // renewed.accessToken and renewed.refreshToken are your new pair
